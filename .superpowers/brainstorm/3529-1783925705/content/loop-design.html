修订版架构 — 循环协作模式

C(状态机) + B(任务队列) 融合 · 闭环迭代直到通过

核心循环

这不是一次性的流水线，而是持续循环——编排→执行→检查→测试，不通过就回到编排重新来，直到项目完全通过。

闭环状态机 (主循环)
                       ┌─────────┐
                       │  IDLE   │
                       └────┬────┘
                            │ 用户输入需求
                            ▼
              ┌─────────────────────────┐
              │   🔄 ORCHESTRATING      │
              │   编排Agent拆分任务       │
              │   → 输出: [(task,agent)] │
              └────────────┬────────────┘
                           │ 脚本分发任务到队列
                           ▼
              ┌─────────────────────────┐
              │   📤 DISPATCHING        │
              │   任务进入各Agent队列     │
              │   ┌─────┐┌─────┐┌─────┐│
              │   │T1→A1││T2→A2││T3→A3││
              │   └──┬──┘└──┬──┘└──┬──┘│
              └──────┼──────┼──────┼───┘
                     │      │      │
                     ▼      ▼      ▼
              ┌─────────────────────────┐
              │   ⚙️ EXECUTING          │
              │   Agent并行执行各自任务   │
              │   完成后通知脚本           │
              │   [████████░░] 进度可见   │
              └────────────┬────────────┘
                           │ 全部完成
                           ▼
              ┌─────────────────────────┐
              │   🔍 REVIEWING          │
              │   检查Agent审核所有结果    │
              └────────┬───────┬────────┘
                       │       │
                  通过  │       │ 不通过 (有问题)
                       │       ▼
                       │  ┌─────────────────────┐
                       │  │ 问题反馈给编排Agent    │
                       │  │ "任务X的输出有问题"    │
                       │  │ → 回到 ORCHESTRATING │
                       │  └─────────────────────┘
                       ▼
              ┌─────────────────────────┐
              │   🧪 TESTING            │
              │   测试Agent验证项目       │
              └────────┬───────┬────────┘
                       │       │
                  通过  │       │ 不通过 (测试失败)
                       │       ▼
                       │  ┌──────────────────────────────┐
                       │  │ 测试失败报告 → 编排Agent       │
                       │  │ "3个测试失败: login, payment"  │
                       │  │ 编排Agent重新分析→重新拆分任务   │
                       │  │ → 回到 ORCHESTRATING           │
                       │  └──────────────────────────────┘
                       ▼
              ┌─────────────────────────┐
              │   ✅ COMPLETED          │
              │   项目产出+完整日志       │
              └─────────────────────────┘

关键设计决策确认

🔄 循环迭代

REVIEWING 不通过 → ORCHESTRATING

TESTING 不通过 → ORCHESTRATING

编排Agent拿到失败原因，重新分析、重新拆分

🛑 终止条件

通过条件: 检查通过 + 测试通过

兜底: 最大迭代次数(如5轮)，超限则报错退出

每轮迭代编号可见

📋 增量任务

返工时只分配需要修复的新任务

已完成且通过检查的任务不重复执行

编排Agent需要知道"哪些已完成"
