基于 ACP 协议的多 Agent 编排框架 · 概念图

核心理念

编排 Agent 只负责拆任务、指派任务、确认结果。执行 Agent 彼此隔离，只做好自己那部分。中间脚本充当 ACP 通信中枢。

关键约束

🔇 Agent 互不通信

执行 Agent 之间完全隔离，只能通过中间脚本中转

📡 纯 ACP 协议

不通过注册配置，直接用 ACP 调用已安装的 Agent

👁️ 全链路可见

所有编排→执行→检查→测试的消息都可查看

📝 脚本定义专长

在脚本中为每个 Agent 写提示词规范，不做全局注册
